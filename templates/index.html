Description

The URL is processed by using N-Gram vectorization and the classification of a Random Forest algorithm (short: RF), that has been trained on 500.000 Sample-URLs with 250.000 malicious blacklisted URLs and 250.000 of the most visited URLs worldwide.

The RF mimics the behaviour of an experienced professional by assessing the potential threat of the URL simply by evaluating its language. Each time another char of the URL is being typed in the field above an AJAX-call is sent to a Flask-API, which governs the trained RF.

Note: This is a POC. The ground truth for this project are public blacklists. If a website you might condemn harmful (e.g. NSFW) but the RF still doesn't classify it as malicious, it is because it hasn't been condemned by blacklist companies as harmful. If you want to include certain URLs as additional malicious subjects, you need to train the RF-model on such a dataset.
